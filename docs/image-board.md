## 1. Design Direction（デザインの方向性）

### コンセプトキーワード

```
┌─────────────────────────────────────────┐
│                                         │
│   プロフェッショナル × 親しみやすさ    │
│                                         │
│   信頼性 × 先進性                      │
│                                         │
│   一貫性（一気通貫）× 柔軟性          │
│                                         │
└─────────────────────────────────────────┘
```

### ビジュアルトーン
- **Clean & Modern**: 不必要な装飾を排除
- **Spacious**: 余白を贅沢に使う
- **Soft**: 硬すぎず、柔らかすぎず
- **Trustworthy**: 企業としての信頼感

## 3. Typography（タイポグラフィ）

### Font: Noto Sans JP

```
見出し（Bold, 36-68px）
AaBbCcDdあいうえお漢字

本文（Light/Regular, 15-18px）
AaBbCcDdあいうえお漢字
読みやすさを重視した行間設定（2.0-2.1）
```

### 参考サイトとの比較

| サイト | フォント | 特徴 |
|--------|----------|------|
| SmartHR | サンセリフ | クリーン、読みやすい |
| LayerX | 明朝+ゴシック混在 | 文字間スペーシングで劇的効果 |
| freee | ゴシック体 | 日本語最適化 |

**→ Noto Sans JPは適切な選択**

## 6. Imagery Style（画像スタイル）

### 現在の課題
❌ サービスセクションがジオメトリックSVGのみ  
❌ 抽象的で具体性に欠ける  
❌ ブランドの個性が弱い

### 理想の方向性

#### オプションA: スクリーンショット + UI
```
┌─────────────────────┐
│                     │
│  [実際のシステム]   │
│  [画面キャプチャ]   │
│                     │
└─────────────────────┘
```
**メリット**: 具体的、信頼感  
**参考**: Sansan（デバイスモックアップ）

#### オプションB: カスタムイラスト
```
┌─────────────────────┐
│                     │
│  [ピンク・グリーン] │
│  [を使った線画]     │
│                     │
└─────────────────────┘
```
**メリット**: ブランド性、親しみやすさ  
**参考**: SmartHR（イラスト混在）

#### オプションC: アイコン + 写真
```
┌─────────────────────┐
│  [大きめアイコン]   │
│  ＋                  │
│  [背景写真ぼかし]   │
└─────────────────────┘
```
**メリット**: バランス、柔軟性

**→ 推奨: オプションB（カスタムイラスト）**  
社内で作成可能、ブランド性確保、親しみやすさ

## 11. Design Principles（デザイン原則）

### Do's ✅
- **余白を贅沢に使う** - SmartHR, LayerX参考
- **情報階層を明確に** - 見出しと本文のサイズ差を大きく
- **カラーは控えめに** - ピンクをアクセントとして戦略的に
- **具体性を持たせる** - 抽象的な説明より、ビジュアルで伝える
- **一貫性を保つ** - コンポーネントの使い方を統一

### Don'ts ❌
- **装飾過多** - 必要最小限のデザイン要素
- **多色使用** - ブランドカラー以外を乱用しない
- **派手なアニメーション** - BtoB SaaSは控えめが基本
- **情報の詰め込み** - スクロールを恐れず、余白で区切る
- **フォント混在** - Noto Sans JPで統一
