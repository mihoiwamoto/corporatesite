## 2. 技術構成

| レイヤー | 内容 |
|---|---|
| フロント | 素の静的HTML + CSS + バニラJS（フレームワーク／ビルドツールなし） |
| CSS | 各ページの `<style>` インライン + 共通CSS（`assets/css/header.css`, `footer.css`, `background.css`, `service-icons.css`） |
| フォント | Noto Sans JP（Google Fonts: 300/400/500/700） |
| 共通コンポーネント | ヘッダー/フッターを JS で動的注入（`assets/js/header.js`, `footer.js`。`<div id="site-header">` / `<div id="site-footer">` をプレースホルダとして使用） |
| アニメーション | IntersectionObserver による `.reveal` スクロール表示、サービスアイコンの Lottie（`dotlottie-player.js` + SVGフォールバック） |
| ホスティング | Vercel（`vercel.json`：`api/*.js` をサーバーレス関数として実行、maxDuration 10s） |
| バックエンド | Supabase（`@supabase/supabase-js`）。`api/feedback.js`・`comments.js`・`notify.js`（Slack通知）・`check-feedback.js` |
| 注意（自動ビルド） | 保存後に詳細ページが `news/` 配下へ、アセットが `/assets/` へ自動再配置・パス正規化される。Edit失敗やファイル消失はこの挙動が原因のことがある |

> 補足: Supabase / api / feedback 系は**サイト本体の公開機能ではなく、社内レビュー（フィードバック収集）用のしくみ**。詳細は §6 参照。

## 4. ページ別構成

### 4-1. トップ `index.html`
上から順に:
1. **FV（ファーストビュー）** — 2カラム。左: 西原商会グループバッジ＋ `h1「業務に溶け込む、システムをつくる。」`＋リード。右: `fv-illustration.svg`（浮遊アニメ）。背景はピンクのグラデーション。
2. **ABOUT** — 2カラム。見出し `「システムでもっと強くする。それが、私たちの仕事です。」`＋本文3段（グループのIT企業/基幹領域/企画〜運用一貫）。
3. **SERVICES** — `事業内容`。3項目を縦並びの大カードで（1.企画・要件定義 / 2.設計・開発 / 3.保守・運用）。各カードにキャッチ・説明・Lottieアイコン・詳細リンク。末尾に「事業内容を詳しく見る」。
4. **GROUP** — `西原商会グループの一員として`。左: グループロゴ、右: 説明＋外部サイトリンク。緑系アクセント。
5. **RECRUIT** — 2カラム。左: `「一緒に、つくりませんか。」`＋「募集職種を見る」CTA。右: 求人カード（システムエンジニア（Web系））。
6. **INTERVIEW** — `社員の声`。インタビューカード（`分業せず、最後まで関わる開発がしたくて。` / 開発部・2023年入社）。
7. **OFFICE GALLERY** — オフィス写真4枚の全幅グリッド（`office-01〜04.png`）。

## 6. 社内レビュー（フィードバック）システム

公開サイトとは別に、社内レビュー用の仕組みが同梱されています。
- `feedback-panel-v2.js`: ページ単位でフィードバックを投稿・追跡する浮遊パネル。
- `review-mode.js`: 要素ハイライト＋クリックでコメント入力するレビューモード。状態は localStorage に保持。
- `review-admin.html`: フィードバック集計・スレッド・ステータス管理（未対応/対応中/確認待ち/完了/保留）。
- `api/feedback.js` `comments.js` `notify.js`（Slack）`check-feedback.js` + Supabase（`api/migrations/*.sql`）。

> 一般公開時にはこれらのFAB／スクリプト読み込みを除去する想定。

## 7. デザインシステム（実装値ベース）

> ⚠️ `DESIGN.md` と**実装に差分あり**。下表は `index.html` 等の実CSS変数を正とする。`DESIGN.md` のピンクは `#E07898`／角丸4px だが、実装は `#D4637E`／角丸12px・ピル多用。デザイン案検討時は実装値を基準にすること。

### カラー（実装 CSS変数）
| 役割 | 値 |
|---|---|
| ピンク（主アクセント） | `#D4637E` / dark `#B84A66` / light `#FDF2F5` / gradient `#F9E4EC` |
| グリーン（グループ専用） | `#1A9D57` / dark `#147A44` / light `#ECF7F1` |
| インク（本文） | `#141C24` |
| サブテキスト | `#4A5568` |
| 背景 | 白 `#ffffff` / soft `#F7F8FA` |
| 罫線 | `#E8ECF0` |

- 白ベース。ピンクは唯一の主アクセント（CTA・eyebrow・hover）。
- 緑は**西原商会グループ関連でのみ**使用。

### タイポグラフィ
- フォント: Noto Sans JP（Latin専用ディスプレイフォントは混在させない）。
- eyebrow: 11〜12px / 600 / ピンク / 大文字 / トラッキング広め（.12〜.25em）。
- FV `h1`: 68px（〜80px@1920）。section `h2.title`: 36px前後（〜44px）。
- 本文: 14〜16px / line-height 1.8〜2.2（日本語の読みやすさ重視）。
- `body` に `text-wrap: pretty` + `word-break: auto-phrase`（改行位置の自然さ重視）。

### レイアウト・形状・陰影
- 最大幅 1200px（1440+で1320px / 1920+で1400px）。左右padding 40px（モバイル20px）。
- section 縦padding 120px（モバイル80px）。`.soft` セクションは薄グレーへのグラデ。
- 角丸: カード12〜24px、ボタン/タグ/バッジは **ピル（9999px）** を多用。
- 影: ごく軽い多段シャドウ（`--shadow-sm〜xl`）。hoverで `translateY` 浮き＋影増し。
- レスポンシブ: 768pxで多カラム→1カラム。タブレット帯は `clamp()` で流体スケール。

### コンポーネント
- `.btn-primary`: ピンク地・白文字・ピル・影。hoverで暗ピンク＋浮き。
- `.btn-ghost`: 透明地・罫線・hoverでピンク。
- `.btn-cta`（CTA帯）: ダーク地・白文字。
- `.card`: 白地・1px罫線・角丸12px・軽い影、hoverで浮き。
- `.eyebrow`: セクション見出し上のピンク・大文字ラベル。
- 絵文字はUIに使わない方針（ただし現状 interview のプレースホルダに 👤 が残存）。
