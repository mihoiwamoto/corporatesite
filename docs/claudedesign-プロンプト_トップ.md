## --- プロンプトここから ---

あなたはBtoB／採用領域に強いシニアWebアートディレクター兼フロントエンド実装者です。日本語コーポレートサイトの**トップページ**のデザイン案を作成してください。既存サイトのリブランドではなく「現行トーンの正統進化（ブラッシュアップ）」がゴールです。

### 1. 会社・前提
- 会社: 株式会社ランステック（Lans Tech）。西原商会グループ（創業140年以上の食品流通グループ）の**社内IT企業**。
- 重要: 案件は**グループ内のみ**で外部クライアント獲得はしない。**外部営業トーン（「お任せください」「実績多数」等の受注狙い表現）は使わない**。
- 中核メッセージ軸: **「企画から運用まで、同じチームが一貫して担う＝分業しない」**。「つくって終わり」ではなく改善し続けて事業に伴走する、を伝える。
- 主要ターゲット: **転職を検討する中途エンジニア（25〜39歳・実務3年以上）**。スカウト経由で必ずこのサイトを見て応募可否を判断する。よって「安定（グループ基盤）× 先進（技術・AI活用）」のハイブリッド像と、働く環境の信頼感を最優先で伝える。
- 文体: 落ち着いて誠実、誇張しない。短く言い切る日本語見出し（1〜2行）。

### 2. デザイン方向（現行を磨く）
現行のデザイントークンを基盤に、完成度・余白設計・階層・モーション品質を引き上げてください。色相そのものは変えず、洗練させる方向。

- カラー（基準値・微調整可）:
  - 主アクセント=モーヴ/ピンク `#D4637E`（dark `#B84A66` / light `#FDF2F5` / グラデ用 `#F9E4EC`）
  - グループ専用グリーン `#1A9D57`（**西原商会グループの言及部分でのみ**使用）
  - インク `#141C24` / サブ `#4A5568` / 背景 白 `#FFFFFF`・ソフト `#F7F8FA` / 罫線 `#E8ECF0`
- 白ベース。ピンクは唯一の主アクセント（CTA・eyebrow・hover・強調）。サチュレーションの高い色を増やさない。
- タイポ: **Noto Sans JP**（300/400/500/700）。Latin専用ディスプレイフォントは混ぜない。セクション見出しの上に「英字eyebrow（大文字・トラッキング広め・ピンク）」。日本語の読みやすさのため line-height 1.8〜2.2、`text-wrap: pretty` / `word-break: auto-phrase` を使う。
- レイアウト: 最大幅1200px・左右余白たっぷり、左揃え基調。白とソフトグレーのセクションを交互にしてリズムを作る。角丸はカード12〜24px／ボタン・タグはピル。影は軽く多段、hoverで `translateY` 浮き＋影。
- モーション: スクロールで `.reveal`（フェード＋上方向）。やりすぎない、上品な範囲。
- 禁止/注意: 装飾的な追加ブランドカラー・グラスやヘビーシャドウ・絵文字をUIに使わない。緑をグループ以外に使わない。

### 3. トップページに必要なセクション（この順番・内容を踏襲しつつ質を上げる）
1. **ファーストビュー** — 西原商会グループのバッジ＋強いh1＋リード＋（任意で抽象ビジュアル）。現行h1は「業務に溶け込む、システムをつくる。」。これを軸に、より刺さる案を1〜2提案してよい（外部営業トーンは禁止）。CTAは「採用情報を見る」を主に、「お問い合わせ」を副に。
2. **About（私たちについて）** — グループのIT企業であること、基幹領域（受発注・在庫・物流・販売管理）、企画〜運用を同じチームで一貫、を簡潔に。
3. **Services（事業内容）** — 3軸「①企画・要件定義 ②設計・開発 ③保守・運用」をライフサイクルとして見せる。各にキャッチ＋短い説明＋アイコン。
4. **Group（西原商会グループ）** — 安定した経営基盤の訴求。緑アクセントはここだけ。外部サイトへの導線。
5. **Recruit（採用）** — 「一緒に、つくりませんか。」。中途エンジニアに向けた一貫して関われる環境の訴求＋「募集職種を見る」CTA＋求人カード（システムエンジニア（Web系））。
6. **Interview（社員の声）** — インタビュー誘導カード（例: 「分業せず、最後まで関わる開発がしたくて。」）。
7. **（任意）オフィス/カルチャー** — 雰囲気が伝わる帯。※写真使用は暫定方針のため、写真ありき／なし（図形・タイポのみ）の両対応を意識。
8. CTA帯（お問い合わせ）→ フッターは今回スコープ外（簡易でよい）。

### 4. 成果物（output）
- **トップページ1枚もの**の、レスポンシブな**HTML+CSS**（ファーストビュー〜CTAまで全セクション）。バニラ実装・外部UIフレームワーク不使用（現行と同じ前提）。フォントは Google Fonts の Noto Sans JP。
- デスクトップ最適＋768pxでのモバイル崩れがないこと。
- 冒頭に `:root` のCSSカスタムプロパティでデザイントークンを定義。
- ダミーコピーは上記の文言・トーンに沿った日本語で。`【要確認】`は使わず、それらしい本文を入れる。
- 最後に**100〜150字で「現行からどこを磨いたか（意図）」**を箇条書きで説明。

### 5. 制約まとめ（守ること）
- 外部受注を狙う営業コピーにしない（社内IT・グループ内案件）。
- ピンク単一アクセント＋グループ緑のみ。Noto Sans JP のみ。絵文字UI禁止。
- 「分業しない／企画〜運用一貫」を必ず軸に据える。
- 写真は暫定方針 → 図形・タイポ・余白で十分成立する設計にする。
